# 余弦相似度（Cosine Similarity）

## 1. 定义

余弦相似度衡量的是两个向量在方向上的接近程度，与向量的长度（模）无关。对于向量
**A** 和 **B**：

```
                A · B
cos(θ) = ----------------------
          ||A|| × ||B||
```

- **A · B**：两向量的点积（对应分量相乘再求和）
- **||A||、||B||**：两向量各自的模（长度）
- **θ**：两向量之间的夹角

取值范围是 **[-1, 1]**：

| cos(θ) | 含义 |
|---|---|
| 1 | 方向完全相同 |
| 0 | 方向正交（无关） |
| -1 | 方向完全相反 |

## 2. 几何意义

余弦相似度只关心「方向」，不关心「长度」。这也是它和欧氏距离最大的区别：一个很长的
向量和一个很短的向量，只要指向同一个方向，余弦相似度依然是 1。下面的动态图里，向量
**B** 的长度会周期性伸缩，但只要它和向量 **A** 的夹角不变，`cos θ` 的数值就不变——
用来直观感受这一点。

**代数视角（逐分量相乘再求和）：**

| 分量 | A | B | A 分量 × B 分量 |
|---|---|---|---|
| x | 4 | 1 | 4 × 1 = 4 |
| y | 1 | 3 | 1 × 3 = 3 |

**A · B = 4 + 3 = 7**

**几何视角（校验两种算法一致）：**

```
|A| = √(4² + 1²) = √17 ≈ 4.123
|B| = √(1² + 3²) = √10 ≈ 3.162

cos θ = A·B / (|A| |B|) = 7 / (4.123 × 3.162) ≈ 0.537   → θ ≈ 57.5°
```

## 6. 与欧氏距离的区别

- **欧氏距离**：关心两点在空间中的绝对位置差，会受向量长度（模）影响。
- **余弦相似度**：只关心方向，不受长度影响——这正是上面动画中 B 的长度反复缩放、但
  `cos θ` 只随夹角变化的原因。

在文本向量、词向量、Embedding 检索等高维稀疏或长度不统一的场景中，余弦相似度通常比
欧氏距离更能反映「语义方向」上的接近程度，这也是 RAG 检索、Agent 语义记忆匹配中最常
用的相似度度量。

## 7. Java 实现示例

```java
public class CosineSimilarity {

    public static double cosineSimilarity(double[] a, double[] b) {
        double dot = 0.0, normA = 0.0, normB = 0.0;
        for (int i = 0; i < a.length; i++) {
            dot += a[i] * b[i];
            normA += a[i] * a[i];
            normB += b[i] * b[i];
        }
        if (normA == 0 || normB == 0) {
            return 0.0;
        }
        return dot / (Math.sqrt(normA) * Math.sqrt(normB));
    }

    public static void main(String[] args) {
        double[] a = {1.0, 2.0, 3.0};
        double[] b = {2.0, 4.0, 6.0}; // 与 a 同方向，长度不同
        System.out.println(cosineSimilarity(a, b)); // -> 1.0
    }
}
```
