灰色段是「旧状态占比 (1-z_t)」，紫色段是「候选值占比 z_t」，两段加起来永远填满整条 进度条——这就是 GRU 用一个更新门把「遗忘」和「写入」耦合在一起的凸组合，不会像 LSTM 那样让两个独立门同时偏大导致数值失控。z_t 越接近 0，新的 h_t 就越接近上一步 的 h_(t-1)，梯度反传时几乎不衰减；z_t 越接近 1，则几乎完全替换成候选值 h̃_t。
