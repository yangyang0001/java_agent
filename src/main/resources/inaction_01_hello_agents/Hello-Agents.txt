4、根据上面的提出的问题, 引申几个核心问题:
    人类为什么要研究智能体? 是要代替人类思考吗? 既然人类有最智能的处理流程, 机器是否能够模拟人类处理问题的方式来处理 人类能够处理的问题!
    人类处理流程:
        环境 ---> 接收环境中的信息 [听觉, 视觉等] ---> 大脑 ---> 给出执行指令 ---> 处理事情

        此流程不断重复!

    Agent 的处理流程：
        Environment --->  Perception  ---> Agent ---> Command ---> Action

        此流程不断重复!

    人类和目前智能体最大的差距: 人类自主决策, Agent 是靠 数据+算法 实现类似于人类的 自主性

5、传统的 智能体 或者 古典智能体的 概念和演进过程:
    假设我们是 Agent 的研究的先驱者, 我们第一步该怎么搞? 既然计算机程序 = 数据结构 + 算法! 我们也会用 传统的 Input ---> 处理程序 ---> Output
    这给我带来一个 最致命的问题: 输入 参数是固定的, 不能灵活处理多参数的问题; 虽然后来能够使用 多个参数的方式 或者 传入 Function 的方式进行处理, 但是绝对满足不了整个世界的环境变化
    这类智能体称之为 反射智能体; 这个是人类基于计算机程序最早的 智能体 模型!

6、给出现在流行的智能体模型: 人类在输入和输出上 做了更细致的划分! 即: 输入的信息可能更加繁琐, 输出的更加符合人类的理性判断!
    基于模型的 反射智能体;
    基于目标的 智能体: 又分为 基于效用的智能体 和 学习型智能体!

7、传统 智能体 和 基于 LLM 的智能体之间的差距! 基本从 输入 决策 输出 都有了质的变化! 具体参考 002_传统智能体和基于LLM模型的智能体之间的区别.png
    基于 大语言模型的 智能体 和 传统智能体的最大区别: 已经不是再关注写代码了, 而是 引导一个 "大脑" 去规划、行动 和 学习!


8、AI范式:
    符号主义AI:    固定规则, 逻辑边界严谨, 往往在 医疗行业中 最为盛行! 必须按照 医疗行业的 规范来做决定!
    亚符号主义AI:  通过大量的训练给出一个结论, 有时候可能不是太准, 但是通过训练和学习能够越来越精确!
    混合符号主义AI: 结合前两者的优点, 既能处理 符号主义AI的 严格行业规则, 又能处理 规则外的一些动态规划的东西!

---------------------------------------------------------------- 第3章 大预言模型基础 ---------------------------------------------------
--------------------------------------------------------------------------------------------------------------------------------------
1、引出一个问题? 现在的基于 LLM 大预言模型的 智能体是如何工作的! 虽然现在智能体符合 PEAS 的标准规范, 但是对内部的一些具体流程是不清晰的, 就像人类解决问题的思路一样
    问题 ---> 人类搜索自己的大脑中有没有类似的或者相关的处理模型 ---> 如果有直接处理掉了
                                                         ---> 如果没有 ---> 学习相关的资料 ---> 建立处理这类问题的模型或方法论 ---> 解决问题

2、既然有语言, 我们所说的所有语言 在 Agent 的解析过程中统称为 词元!

3、既然有人类思考问题的方式了, LLM Agent 的难点就是: 去哪里找相关的学习资料? 找什么样的学习资料? 找到后如何学习和训练? 强化训练解决问题的能力如何?

4、LLM 语言模型处理人类语言的提示词相关的东西: 从数学角度来说  马尔可夫假设(链式法则) ---> 向量化(余弦相似度)[这块需要矩阵转换, 生成N维向量或坐标]

5、RNN: 循环神经网络; LSTM: 长短时间记忆网络; GRU: 门控循环单元; Transformer: 大语言模型!

------------------------------------------------------------- 第4章 智能体经典范式构建 --------------------------------------------------
--------------------------------------------------------------------------------------------------------------------------------------
1、为了更好的理解 智能体 的 思考和行动, 出现了经典的架构范式! 通过历史和现状而言 无论再经典的 架构范式 都必须 有输入和输出, 即 有思考和结果!
Reasoning and acting    : 思考和行动机密结合的范式, 目的: 让智能体边想边做, 随时动态调整!               边思考边实践
Plan and slove          : 做计划 然后 解决问题; 有一种完整的计划然后执行的思想架构理念!                 先做计划, 根据计划进行实践
Reflection              : 反思范式, 这种范式 让 Agent 能够根据 经验 甚至 未知的结果 给自己一种 反馈!    不断反思

2、为什么要讲这一节课呢? 目的: 我们通过 了解经典架构范式 的思考问题的方式, 出现错误的时候 经典架构是如何处理的, 例如: 调用工具的重试机制, 出现循环调用的时如何处理的?
   这让开发者能从 最经典的架构范式 中学习到 比较优秀的 处理方式和思维范式, 对 Agent 开发者也是一个比较好的 指导!
