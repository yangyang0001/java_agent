# 学习建议：看不懂 LLM 底层算法，要不要继续深挖？该不该转 Agent 开发？

## 结论

**不需要死磕到完全弄懂数学细节才能往下走，转 Agent 开发是合适的。**

Agent 开发更多是"把 LLM 当黑盒用"：写 prompt、设计工具调用、编排多步流程、处理
上下文/记忆管理——这些跟 Transformer 内部矩阵怎么算关系不大。

从 `chapter_03`（RNN/LSTM/GRU/Transformer 手推公式、Java/Python 实现、动态演示）
的学习进度看，对底层原理的理解已经比大多数直接上手做 Agent 的人扎实得多，没必要
因为"还没完全吃透"就止步不前。

## 真正的取舍

不懂内部机制，会失去一些"直觉"——比如：

- 为什么长上下文贵（`O(n²)`）
- 为什么模型会在长文档里"失焦"
- 为什么某些 prompt 技巧有效

这些直觉能帮助更好地判断 Agent 该怎么设计、出问题时怎么排查。但这些直觉**可以在
做 Agent 的过程中慢慢补**，不必是前置门槛。

## 建议的节奏

先转 Agent 开发上手实践，遇到具体"为什么"再回来查对应的原理——比如遇到上下文
超长报错，再回头看 `Transformer.md` §6"Transformer 如何记忆"。边做边补比线性推
进更高效。
